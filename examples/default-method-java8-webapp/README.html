Jersey Java8 Default Method Example

This example demonstrates the usage of Java8's new construct - the default methods in interfaces. Default methods can be used as common JAX-RS resource methods.

The application is common web application, only combines methods and interfaces with default methods to show to show that this use case is possible.

Contents

The mapping of the URI path space is presented in the following table:

URI path Resource class HTTP method description
/default-method/application.wadl Generated WADL GET returns XML with the application api description
/default-method/compoundResource InterfaceResource GET returns interface-root in text/plain
/default-method/compoundResource/path InterfaceResource GET returns interface-path in text/plain
/default-method/compoundResource/class ClassResource GET returns class in text/plain

Application is Servlet 3 based, web.xml-less. Everything needed (resources/providers) is registered in the DefaultMethodApplication class.

Running the Example

Run the example as follows:

mvn clean package jetty:run

This deploys current example using Jetty. You can access the application at:

  • http://localhost:8080/default-method/application.wadl
  • http://localhost:8080/default-method/compoundResource
  • http://localhost:8080/default-method/compoundResource/path
  • http://localhost:8080/default-method/compoundResource/class
